# Agent SEO Analyst Reference

## 네이버 SERP 모듈 기반 키워드 분류

### 적용 배경

네이버 키워드 판단은 더 이상 `블로그탭 있음/없음`만으로 결정하지 않는다. SEO Analyst는 모바일 통합검색 첫 화면에서 어떤 모듈이 먼저 노출되는지를 우선 확인하고, 그 결과를 전략 수립과 글쓰기 구조에 전달한다.

공식 확인된 변화:

- 2025년 3월 27일 네이버 AI 브리핑 전체 사용자 출시. 네이버 공식 보도자료 기준으로 공식형/멀티출처형, 숏텐츠형, 플레이스형, 쇼핑형 등 유형별 UX가 적용된다.  
  출처: https://navercorp.com/media/pressReleasesDetail?seq=32374
- 2026년 4월 28일 네이버 AI탭 베타 출시. 네이버플러스 멤버십 사용자를 대상으로 검색, 쇼핑, 로컬 등 버티컬 연결을 강화한 대화형 검색 경험을 제공한다.  
  출처: https://www.navercorp.com/media/pressReleasesDetail?seq=34984
- 2026년 5월 28일 전후 네이버 콘텐츠/AI 검색 전략 발표에서 AI탭 확대, AI 브리핑과 콘텐츠 노출 강화 흐름이 확인된다.  
  출처: https://www.chosun.com/economy/tech_it/2026/05/28/VJXI7J356VGSVGPOCGHHDXEJPQ/

운영 가설:

- 블로그탭 유무는 보조 신호다.
- 핵심 판단 기준은 모바일 통합검색 첫 화면의 첫 블록이다.
- 블로그 글은 블로그/뷰 영역뿐 아니라 AI 브리핑 인용, 플레이스 보조 문서, 클립 보조 텍스트로도 역할을 가질 수 있다.

### SERP 모듈 분류값

SEO Analyst는 키워드를 아래 5개 모듈 중 하나로 분류한다.

| serpModule | 첫 화면 신호 | 검색 의도 | 기본 대응 |
| --- | --- | --- | --- |
| `ai_briefing` | AI 브리핑 텍스트 요약 | 정보 탐색, 비교, 판단 | AEO 오프닝과 FAQ 중심 글 |
| `clip` | 클립/숏폼 영상 | 체감, 시연, 후기 | 클립 병행, 블로그는 보조 텍스트 |
| `place` | 지도, 매장, 플레이스 | 로컬 방문, 상담 | 스마트플레이스 보조 + 방문 전 안내 글 |
| `shopping` | 쇼핑 상품, 가격, 구매 버튼 | 구매 전환 | 쇼핑/스토어 우선, 블로그는 보조 |
| `blog_view` | 블로그, 카페, 포스트 문서 | 롱폼 정보, 문제 해결, 사용법 | 기존 DIA/C-Rank형 블로그 글 |

### 판정 순서

1. 모바일 네이버 통합검색 첫 블록을 확인한다.
2. 첫 블록을 `ai_briefing`, `clip`, `place`, `shopping`, `blog_view` 중 하나로 분류한다.
3. 직접 확인이 어려우면 키워드 언어 패턴으로 임시 예측한다.
4. 연관 검색어와 함께 많이 찾는 방향을 보조 신호로 본다.
5. 블로그탭/뷰탭 유무는 마지막 보조 신호로만 사용한다.

### 언어 패턴 기반 임시 예측

| 패턴 | 신호 키워드 | 예상 serpModule |
| --- | --- | --- |
| 정보/비교형 | 차이, 비교, 어떤, 뭐가 나을까, 이유, 정리 | `ai_briefing` |
| 체감/시연형 | 후기, 리뷰, 실사용, 언박싱, 맛, 연기량, 체감 | `clip` |
| 로컬 방문형 | 지역명 + 전자담배/베이프샵/매장/상담 | `place` |
| 구매 전환형 | 가격, 구매, 살 곳, 최저가, 제품명 단독 | `shopping` |
| 문제 해결형 | 고장, 탄맛, 누수, 안됨, 원인, 해결, 증상 | `blog_view` |
| 사용법/팁형 | 교체 시기, 설정, 사용법, 방법, 관리 | `blog_view` |

### SEO Analyst 출력 필드

SEO Analyst는 기존 검색의도 결과에 아래 필드를 추가한다.

```json
{
  "primarySearchIntent": "비교형",
  "serpModule": "ai_briefing",
  "serpModuleReason": "모바일 통합검색 첫 블록이 AI 브리핑이고 비교/판단형 질의입니다.",
  "serpModuleOptions": ["ai_briefing", "clip", "place", "shopping", "blog_view"],
  "recommendedBlogRole": "B",
  "contentType": "aeo_comparison",
  "aeoTarget": true,
  "blogTabIsPrimarySignal": false,
  "blogTabExists": false,
  "clipCompetition": false
}
```

필드 의미:

- `primarySearchIntent`: 정보형, 비교형, 방문형, 구매검토형, 문제해결형, 사용법형 등 실제 검색 의도
- `serpModule`: 최종 SERP 모듈 분류
- `serpModuleReason`: 판정 근거
- `recommendedBlogRole`: A/B/C/D/E 중 우선 배정 블로그
- `contentType`: writer가 사용할 글 구조 타입
- `blogTabIsPrimarySignal`: 항상 `false`를 기본값으로 둔다

### 블로그 역할 배정

| serpModule | 우선 블로그 | 이유 |
| --- | --- | --- |
| `ai_briefing` | B 또는 C | 비교/판단은 B, 구조/정보 설명은 C |
| `clip` | E | 후기, 체감, 실용 보조 텍스트 |
| `place` | A | 지역 허브, 방문 전 안내, 플레이스 보조 |
| `shopping` | A 또는 B | 구매 전환 보조, 스토어/매장 연결 |
| `blog_view` 문제해결 | D | 증상, 원인, 해결 중심 |
| `blog_view` 사용법 | C | 사용법, 설정, 구조 설명 |

### 모듈별 Writer 지침

#### `ai_briefing`

- 첫 문단에 직접 답변형 한 문장을 배치한다.
- 질문형 또는 구분형 제목을 우선한다.
- FAQ 2~3개를 하단에 둔다.
- `~란 무엇인가`, `완벽 정리`, `총정리` 같은 정의형 제목은 피한다.
- AI 브리핑이 본문을 그대로 대체하지 못하도록 실제 상담 맥락과 판단 기준을 넣는다.

#### `clip`

- 블로그 단독 상위노출을 목표로 잡지 않는다.
- 클립 영상 또는 시연 콘텐츠를 보조하는 설명 글로 쓴다.
- 체감, 후기, 사용 장면, 비교 포인트를 짧고 명확하게 정리한다.
- E블로그 실용/후기형 배정을 우선한다.

#### `place`

- 매장 위치, 주차, 영업시간, 방문 전 확인 사항을 포함한다.
- 스마트플레이스 CTA를 넣는다.
- 글 목표는 블로그 단독 상위노출보다 플레이스 보조 문서 노출이다.
- A블로그 지역 허브형 배정을 우선한다.

#### `shopping`

- 제품 가격/구매 전환 키워드는 쇼핑/스토어가 우선임을 전제로 한다.
- 블로그는 구매 전 확인 기준, 비교 기준, 매장 상담 연결을 보조한다.
- 단정적인 최저가 표현은 피한다.

#### `blog_view`

- 문제해결형은 증상, 원인, 해결 3단계 구조를 유지한다.
- 사용법형은 설정, 교체, 관리 기준을 단계별로 쓴다.
- 기존 DIA/C-Rank 기준, 실제 경험, 내부링크 구조를 함께 적용한다.

### 만수르베이프 전자담배 예시

전자담배 예시는 업종 전용 규칙이 아니라, 모듈 판정을 이해하기 위한 참고 사례다. 실제 구현은 다른 업종에도 적용 가능한 범용 로직으로 유지한다.

#### `ai_briefing` 후보

- 전자담배 입호흡 폐호흡 차이
- 전자담배 처음 살 때 뭐가 좋을까
- 액상 니코틴 농도 차이
- 입문자 전자담배 추천
- 전자담배 vs 궐련 차이
- 전자담배 액상 성분
- 코일 종류 차이
- 베이프 용어 정리

#### `clip` 후보

- 전자담배 연기량 비교
- 액상 맛 후기
- 전자담배 언박싱
- 제품명 + 후기

#### `place` 후보

- 인천 전자담배
- 부평 전자담배
- 만수동 전자담배
- 구월동 전자담배
- 부평 베이프샵
- 인천 전자담배 가게 추천
- 부평 전자담배 상담

#### `blog_view` 후보

- 전자담배 코일 탄맛 원인
- 전자담배 액상 누수 해결
- 전자담배 안켜짐 원인
- 전자담배 코일 교체 시기
- 전자담배 와트 설정 방법
- 전자담배 액상 분리현상
- 전자담배 충전 안됨
- 코일 번 방지법

### 탭 구분 보조 지식

| 탭 | 포함 콘텐츠 | 의미 |
| --- | --- | --- |
| 블로그탭 | 블로그 전용 | 블로그 단독 경쟁 구조 가능 |
| 뷰탭 | 블로그, 카페, 포스트 통합 | 블로그만 경쟁자가 아님 |
| 탭 없음 | AI 브리핑, 클립, 플레이스, 쇼핑 등이 점유 | 해당 모듈 전략으로 대응 |

### 빠른 판별 카드

```text
키워드 입력
  ↓
모바일 통합검색 첫 블록 확인
  ├─ AI 브리핑 → AEO 구조로 작성, B/C 블로그
  ├─ 클립 영상 → 클립 병행, E 블로그 보조
  ├─ 지도/플레이스 → 방문 전 안내 글, A 블로그
  ├─ 쇼핑 → 스토어/구매 전환 보조
  └─ 블로그/카페 문서 → DIA/C-Rank 기준, C/D 블로그
```

## 추가 운영 기준: 관측값, 예측값, 인용 타입 분리

### observedSerpModule / predictedSerpModule

AI 브리핑은 모든 사용자에게 제공되지만 모든 질의에 항상 노출되는 것은 아니다. 따라서 SEO Analyst는 실제 확인값과 언어 패턴 기반 예측값을 분리한다.

```json
{
  "observedSerpModule": "ai_briefing",
  "predictedSerpModule": "ai_briefing",
  "serpModule": "ai_briefing",
  "serpModuleConfidence": "high",
  "serpCheckedAt": "2026-06-09T00:00:00+09:00",
  "serpCheckDevice": "mobile"
}
```

판정 원칙:

- `observedSerpModule`: 실제 모바일 통합검색 첫 화면에서 확인한 모듈
- `predictedSerpModule`: 직접 확인 전 키워드 패턴으로 예측한 모듈
- `serpModule`: 전략에 사용할 최종 모듈. 실제 확인값이 있으면 `observedSerpModule`을 우선한다.
- `serpModuleConfidence`: `high | medium | low`
- 직접 확인 없이 언어 패턴만으로 `ai_briefing`을 확정하지 않는다.

### AI 브리핑 인용 타입

`serpModule = "ai_briefing"`인 키워드는 다시 정보형 인용 우선 타입과 SEO 상위노출 선행 타입으로 나눈다.

```json
{
  "serpModule": "ai_briefing",
  "aiBriefingCitationType": "geo_priority",
  "aiBriefingCitationNote": "정보형 질의는 즉답형 오프닝, FAQ, 파싱 가능한 단락 구조를 우선한다."
}
```

타입:

- `geo_priority`: 정보성, 정의형, 방법형, 문제해결형. 기존 검색 Top10 밖 문서도 AI 브리핑 출처가 될 수 있다는 외부 실측 사례가 있으므로, 문장 구조와 답변 명확성을 우선한다.
- `seo_required`: 비교형, 추천형, 상업형, 구매검토형. 기존 블로그/뷰 상위노출과 AI 브리핑 인용이 더 강하게 연결될 가능성이 있으므로, DIA/C-Rank형 SEO와 AEO 구조를 함께 적용한다.

판별 기준:

| 키워드 패턴 | aiBriefingCitationType |
| --- | --- |
| `~란`, `~이란`, `~원리`, `~성분`, `~구조`, `~이유` | `geo_priority` |
| `~방법`, `~하는 법`, `~팁`, `~시기` | `geo_priority` |
| `~고장`, `~탄맛`, `~누수`, `~안됨`, `~증상` | `geo_priority` |
| `~차이`, `~비교`, `~vs`, `~뭐가 나을까`, `~어떤 게 좋아` | `seo_required` |
| `~추천`, `~입문`, `~처음`, `~골라줘`, `~베스트`, `TOP` | `seo_required` |

주의:

- 위 구분은 네이버 공식 랭킹 알고리즘이 아니라 외부 실측과 운영 경험 기반의 전략 분기다.
- 네이버 메이트 선정 기준을 AI 브리핑 랭킹 알고리즘과 동일하다고 단정하지 않는다.
- 공식적으로 확인 가능한 사실은 네이버 메이트가 AI 브리핑 인용 수를 주요 공개 기준으로 삼는다는 점이다.

### AI 브리핑형 글의 출처 문장 조건

공식형/멀티출처형 AI 브리핑은 다양한 웹문서를 분석하여 답변과 출처를 요약한다. 따라서 AI 브리핑형 글은 FAQ만 붙이는 방식보다, 본문 초반에 출처로 뽑히기 쉬운 독립 문장을 배치해야 한다.

첫 3문단 안에 포함할 것:

1. 질문에 대한 직접 답변 1문장
2. 비교 또는 판단 기준 1문장
3. 예외, 주의, 선택 조건 1문장

예:

```text
입호흡과 폐호흡의 가장 큰 차이는 연기를 입에 머금고 넘기느냐, 바로 폐로 들이마시느냐입니다.
연초 대체감을 원하면 입호흡이 맞고, 큰 연무량과 개방감을 원하면 폐호흡이 더 가깝습니다.
다만 니코틴 농도와 기기 출력이 맞지 않으면 같은 방식이어도 만족감이 크게 달라질 수 있습니다.
```

### 플레이스형 세분화

`serpModule = "place"`는 하나로 묶지 않고 하위 타입을 둔다.

| placeSubtype | 예시 | 전략 |
| --- | --- | --- |
| `place_city` | 인천 전자담배, 부평 전자담배 | 지역 허브, 매장 선택 기준, 플레이스 CTA |
| `place_dong` | 만수동 전자담배, 구월동 전자담배 | 동 단위 방문 안내, 가까운 상담 맥락 |
| `place_station` | 만수역 전자담배, 부평역 전자담배 | 역세권 방문 동선, 퇴근길/이동 중 방문 팁 |
| `place_visit_check` | 부평 전자담배 상담, 전자담배 매장 방문 전 | 방문 전 체크리스트, 상담 준비물 |
| `place_review` | 부평 전자담배 후기, 만수동 전자담배 리뷰 | 블로그 리뷰/방문자 리뷰와 연결되는 경험 서술 |

플레이스형 글에는 블로그 본문과 함께 스마트플레이스 보강 체크리스트를 생성한다.

체크리스트:

- 대표키워드가 실제 서비스/상품/업종을 설명하는가
- 주소가 동/역세권 검색과 충돌하지 않는가
- 영업시간, 주차, 결제수단, 톡톡, 쿠폰, 예약/주문 정보가 최신인가
- 업체 사진과 대표 사진이 충분한가
- 블로그 리뷰가 지도 정보를 포함해 자동 추출될 수 있는가
- 방문자 리뷰 키워드와 실제 상담 강점이 맞는가

### AI탭 대비 메모

AI탭은 대화형 검색 구조이므로, 한 번 인용된 콘텐츠가 후속 대화에서 반복 노출될 가능성이 있다. 현재는 별도 `serpModule = "ai_tab"`을 구현하지 않고 메모 수준으로 둔다.

운영 가설:

- AI 브리핑에 유효한 즉답형 오프닝, FAQ, 파싱 가능한 단락 구조는 AI탭에도 유효할 가능성이 높다.
- 하반기 네이버 AI탭 확대 상황을 보고 `serpModuleOptions`에 `ai_tab` 추가를 검토한다.
