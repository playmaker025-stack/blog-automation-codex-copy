# Agent Review QA Reference

## 목적

Review QA는 writer 결과가 검색 모듈 전략과 사용자 요구사항을 실제로 지켰는지 발행 전에 검수한다. 이 문서는 네이버 SERP 모듈 전략 중 AI 브리핑, 블로그/뷰, 플레이스 대응을 중심으로 한다.

## LAYER 4: AI 브리핑 모듈 전략 검수

기존의 `AI 브리핑 회피` 검수는 모든 글에 일괄 적용하지 않는다. `serpModule`에 따라 회피와 인용 유도를 분리한다.

### 분기 기준

```text
IF serpModule == "ai_briefing":
  → AI 브리핑 인용 유도 구조를 검수한다.

IF serpModule == "blog_view" OR serpModule == "place":
  → AI 브리핑 요약 회피 구조를 검수한다.

IF serpModule == "clip" OR serpModule == "shopping":
  → 블로그 단독 상위노출보다 보조 역할 적합성을 검수한다.
```

### AI 브리핑 인용 유도 검수

필수:

- 첫 3문단 안에 직접 답변, 판단 기준, 예외 조건이 각각 독립 문장으로 존재한다.
- FAQ 2~3개가 실제 검색 질문 형태로 존재한다.
- 목록, 단계, 표 중 하나 이상이 있어 AI가 파싱하기 쉽다.
- 근거 없는 수치와 단정적 표현이 없다.

판정:

- `적합`: 직접 답변, FAQ, 파싱 구조, 신뢰도 조건을 모두 충족
- `보완 필요`: 직접 답변은 있으나 FAQ 또는 구조화가 약함
- `부적합`: 일반론이 많고 인용 가능한 답변 문장이 없음

### AI 브리핑 요약 회피 검수

`blog_view` 또는 `place` 글은 AI가 본문을 통째로 요약해 클릭을 대체하지 않도록, 요약 가능한 백과사전식 글을 피한다.

검수 기준:

- 제목이 `~란 무엇인가`, `총정리`, `완벽정리`처럼 정의형으로만 끝나지 않는다.
- 첫 문단에서 모든 답을 끝내지 않는다.
- 실제 경험, 매장 상담, 비교 맥락, 방문 전 확인 포인트가 포함된다.
- 내부링크 또는 방문 CTA가 글 후반부에 자연스럽게 이어진다.

## LAYER 6: AI 브리핑 인용 가능성 검수

네이버 메이트는 AI 브리핑 인용 수를 주요 공개 기준으로 삼는다. 단, 아래 기준을 AI 브리핑 랭킹 알고리즘과 동일하다고 단정하지 않는다. Review QA에서는 운영 지표로만 활용한다.

### 6가지 점검 기준

| 항목 | 검수 내용 | 판정 |
| --- | --- | --- |
| AI 브리핑 인용 구조 | 즉답형 오프닝, FAQ, 파싱 가능한 목록/단계/표 | 적합 / 보완 필요 / 부적합 |
| 주제 전문성 | 경험 기반 서술, 업계 용어, 구분/판단 기준 | 높음 / 중간 / 낮음 |
| 서비스 활동성 | 최근 발행 주기, 이웃/커뮤니티 활동 | 정상 / 점검 필요 |
| 콘텐츠 신뢰도 | 근거 없는 수치, 과장 표현, 확인 필요 정보 여부 | 높음 / 중간 / 낮음 |
| 이용자 반응 유도 | 공감/댓글을 유도하는 질문형 마무리, 경험 연결 | 있음 / 없음 |
| 검색 활용도 | 제목과 소제목이 실제 검색 쿼리와 맞는지 | 높음 / 중간 / 낮음 |

종합 판정:

```text
AI 브리핑 인용 가능성: 높음 / 중간 / 낮음
차단 사유:
- 신뢰도 낮음
- 직접 답변 없음
- FAQ 없음
- 검색 쿼리와 제목 불일치
```

## 금지 표현과 위험 패턴

본문에는 내부 제작용 용어를 쓰지 않는다.

- AI 브리핑 인용 가능성
- SEO 점수
- SERP 모듈
- 메인 키워드
- 서브 키워드
- 검색 활용도
- 상위노출용
- 내부링크 설계

위 용어는 리포트에는 쓸 수 있지만 발행 본문에는 쓰지 않는다.
