# Project Principles

## Verification-First Rule

This project must not present unverified or uninspected facts as confirmed facts.

Required behavior:

- Inspect the exact file before describing code behavior.
- Separate every non-trivial claim into one of:
  - `Confirmed`
  - `Inferred`
  - `Unknown`
- If a fact is not verified, say that it is not verified.
- Do not fill gaps with plausible-sounding details.
- If later evidence contradicts an earlier claim, correct it explicitly and immediately.

This rule overrides speed, convenience, and persuasive wording.
